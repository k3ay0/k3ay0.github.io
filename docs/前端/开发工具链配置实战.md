# 开发工具链配置实战

记录开发过程中遇到的工具配置问题。

## pnpm全局目录配置

pnpm全局安装的包找不到命令，提示bin目录不在PATH中。

解决方法是设置pnpm的全局目录：
```bash
pnpm config set global-bin-dir ~/.local/bin
```

然后把~/.local/bin加到PATH里。

## SSH连接GitHub

SSH连接GitHub失败，常见原因有几个：

1. SSH密钥没配置好。用ssh-keygen生成密钥，然后把公钥添加到GitHub。
2. SSH配置文件有问题。检查~/.ssh/config：
```
Host github.com
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_rsa
```
3. 网络问题。国内访问GitHub偶尔不稳定，可以考虑用代理。

## Git基础配置

新环境的Git配置不能忘：
```bash
git config --global user.name "Your Name"
git config --global user.email "your@email.com"
```

分支命名规范、commit message格式，团队最好统一标准。

## 总结

工具配置是开发的基础，遇到问题别急，慢慢排查总能找到原因。
