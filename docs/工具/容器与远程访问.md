# 容器与远程访问实战：Docker与Tailscale

容器技术和远程访问工具在现代开发中越来越重要。Docker让应用部署更简单，Tailscale则提供了零配置的VPN体验。

本文将介绍这两个工具的使用方法和常见问题的解决方案。

## Docker基础操作

Docker是容器化的标准工具，让应用打包和部署变得简单。

**基本概念**

- 镜像（Image）：应用的只读模板
- 容器（Container）：镜像的运行实例
- Dockerfile：构建镜像的脚本
- docker-compose：多容器编排

**常用命令**

```bash
# 镜像操作
docker pull nginx  # 拉取镜像
docker images  # 列出镜像
docker rmi nginx  # 删除镜像

# 容器操作
docker run -d --name web -p 80:80 nginx  # 运行容器
docker ps  # 查看运行中的容器
docker ps -a  # 查看所有容器
docker stop web  # 停止容器
docker rm web  # 删除容器

# 进入容器
docker exec -it web /bin/bash

# 查看日志
docker logs -f web
```

**Dockerfile编写**

```dockerfile
# Node.js应用示例
FROM node:18-alpine

WORKDIR /app

COPY package*.json ./
RUN npm ci --only=production

COPY . .

EXPOSE 3000
CMD ["node", "app.js"]
```

**docker-compose**

```yaml
# docker-compose.yml
version: '3.8'

services:
  web:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
    depends_on:
      - db
  
  db:
    image: postgres:14
    environment:
      - POSTGRES_DB=myapp
      - POSTGRES_PASSWORD=secret
    volumes:
      - pgdata:/var/lib/postgresql/data

volumes:
  pgdata:
```

```bash
# 启动所有服务
docker-compose up -d

# 查看状态
docker-compose ps

# 停止并删除
docker-compose down
```

## Docker网络与存储

Docker的网络和存储管理是容器化的重要部分。

**网络模式**

```bash
# bridge（默认）
docker run --network bridge nginx

# host
docker run --network host nginx

# none
docker run --network none nginx

# 自定义网络
docker network create mynet
docker run --network mynet nginx
```

**数据持久化**

```bash
# 命名卷
docker volume create mydata
docker run -v mydata:/app/data nginx

# 绑定挂载
docker run -v /host/path:/container/path nginx

# 查看卷
docker volume ls
docker volume inspect mydata
```

**常见问题**

问题1：容器网络不通
```bash
# 检查网络配置
docker network ls
docker network inspect bridge

# 检查容器状态
docker ps
docker logs container_name
```

问题2：数据丢失
```bash
# 确保使用了卷
docker inspect container_name | grep -A 5 Mounts

# 检查卷状态
docker volume ls
```

问题3：镜像拉取失败
```bash
# 配置镜像加速器
# /etc/docker/daemon.json
{
  "registry-mirrors": ["https://mirror.ccs.tencentyun.com"]
}

sudo systemctl restart docker
```

## Tailscale零配置VPN

Tailscale是基于WireGuard的零配置VPN工具，让设备之间的连接变得简单。

**安装和使用**

```bash
# 安装
curl -fsSL https://tailscale.com/install.sh | sh

# 启动
sudo tailscale up

# 查看状态
tailscale status

# 查看IP
tailscale ip
```

**工作原理**

Tailscale使用WireGuard协议，但通过协调服务器自动处理密钥交换和NAT穿透。

设备会自动组网，不需要手动配置IP和路由。每个设备都有一个100.x.x.x的IP地址。

**常见用法**

1. 远程访问家里的服务器
2. 团队成员之间共享服务
3. 访问云服务器而不需要暴露公网端口

```bash
# 连接到远程设备
ssh user@100.x.x.x

# 通过浏览器访问服务
# 如果远程设备运行了Web服务，可以直接访问
```

**容器中使用Tailscale**

```bash
# 运行Tailscale容器
docker run -d   --name=tailscale   --cap-add=NET_ADMIN   --cap-add=NET_RAW   -v /dev/net/tun:/dev/net/tun   -v tailscale-state:/var/lib/tailscale   tailscale/tailscale:latest
```

**常见问题**

问题1：网卡检测失败
```bash
# 检查TUN设备
ls -la /dev/net/tun

# 手动创建
sudo mkdir -p /dev/net
sudo mknod /dev/net/tun c 10 200
```

问题2：连接不稳定
```bash
# 检查日志
journalctl -u tailscaled

# 重启服务
sudo systemctl restart tailscaled
```

问题3：性能问题
```bash
# 检查MTU
tailscale netcheck

# 调整MTU
sudo ip link set tailscale0 mtu 1280
```

## 容器与远程访问最佳实践

一些使用容器和远程访问工具的最佳实践。

**Docker最佳实践**

1. 使用多阶段构建减小镜像体积
2. 不要在镜像中存储敏感信息
3. 使用.dockerignore排除不需要的文件
4. 定期清理无用的镜像和容器

```bash
# 清理无用资源
docker system prune

# 查看磁盘使用
docker system df
```

**安全建议**

1. 不要以root用户运行容器
2. 使用只读文件系统
3. 限制容器资源
4. 定期扫描镜像漏洞

```bash
# 使用非root用户
RUN adduser -D myuser
USER myuser

# 限制资源
docker run --memory=512m --cpus=1 nginx
```

**Tailscale最佳实践**

1. 启用多因素认证
2. 定期审查设备列表
3. 使用ACL控制访问权限
4. 监控连接日志

```json
// ACL配置示例
{
  "acls": [
    {
      "action": "accept",
      "src": ["group:engineering"],
      "dst": ["tag:server:*"]
    }
  ]
}
```

**监控和日志**

```bash
# Docker日志
docker logs --tail=100 container_name

# Tailscale日志
tailscale status --json
journalctl -u tailscaled
```

容器和远程访问工具能大幅提升开发效率，值得花时间学习和配置。
