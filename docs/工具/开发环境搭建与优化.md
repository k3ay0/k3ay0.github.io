# 开发环境搭建指南：Nginx与Python虚拟环境

开发环境的搭建是项目启动的第一步。Nginx是常用的Web服务器，Python虚拟环境则帮助管理项目依赖。

本文将介绍这些工具的配置方法和常见问题的解决方案。

## Nginx配置与管理

Nginx是高性能的Web服务器和反向代理，广泛用于生产环境。

**安装**

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install nginx

# CentOS/RHEL
sudo yum install nginx

# 启动
sudo systemctl start nginx
sudo systemctl enable nginx
```

**基本配置**

```nginx
# /etc/nginx/nginx.conf
worker_processes auto;
events {
    worker_connections 1024;
}

http {
    include       mime.types;
    default_type  application/octet-stream;
    
    sendfile on;
    keepalive_timeout 65;
    
    include /etc/nginx/conf.d/*.conf;
}
```

**虚拟主机配置**

```nginx
# /etc/nginx/conf.d/mysite.conf
server {
    listen 80;
    server_name example.com;
    root /var/www/html;
    
    location / {
        try_files $uri $uri/ =404;
    }
    
    location /api {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

**常见配置错误**

1. 路径配置错误：确保root路径指向正确的目录
2. 权限问题：nginx用户需要有读取文件的权限
3. 端口冲突：检查80端口是否被占用

```bash
# 检查配置语法
nginx -t

# 重新加载配置
nginx -s reload
```

**HTTPS配置**

```nginx
server {
    listen 443 ssl;
    server_name example.com;
    
    ssl_certificate /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;
    
    # 重定向HTTP到HTTPS
    return 301 https://$server_name$request_uri;
}
```

## Python虚拟环境管理

Python项目应该使用虚拟环境隔离依赖，避免不同项目之间的版本冲突。

**venv（Python内置）**

```bash
# 创建虚拟环境
python3 -m venv myenv

# 激活虚拟环境
source myenv/bin/activate

# 退出虚拟环境
deactivate

# 安装依赖
pip install flask

# 导出依赖
pip freeze > requirements.txt

# 安装依赖
pip install -r requirements.txt
```

**virtualenv**

virtualenv是第三方工具，功能比venv更丰富：

```bash
# 安装
pip install virtualenv

# 创建虚拟环境
virtualenv myenv

# 指定Python版本
virtualenv -p /usr/bin/python3.8 myenv
```

**conda**

conda适合数据科学项目：

```bash
# 创建环境
conda create -n myenv python=3.9

# 激活环境
conda activate myenv

# 安装包
conda install numpy pandas

# 导出环境
conda env export > environment.yml
```

**常见问题**

问题1：安装C扩展包失败
```bash
# 缺少编译工具
sudo apt install python3-dev build-essential

# 缺少依赖库
sudo apt install libffi-dev libssl-dev
```

问题2：虚拟环境激活失败
```bash
# 检查shell类型
echo $SHELL

# 使用正确的激活脚本
# bash/zsh
source myenv/bin/activate
# fish
source myenv/bin/activate.fish
```

**最佳实践**

1. 每个项目使用独立的虚拟环境
2. 使用requirements.txt或environment.yml管理依赖
3. 不要把虚拟环境提交到版本控制
4. 使用pipenv或poetry等现代工具

## 其他开发工具配置

一些常用的开发工具配置技巧。

**Git配置**

```bash
# 基本配置
git config --global user.name "Your Name"
git config --global user.email "your@email.com"

# 设置默认编辑器
git config --global core.editor code --wait

# 设置换行符处理
git config --global core.autocrlf true

# 启用颜色输出
git config --global color.ui auto
```

**VS Code配置**

```json
// settings.json
{

  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true

  "python.defaultInterpreterPath": "${workspaceFolder}/.venv/bin/python"

```

**终端工具**

```bash
# 安装常用工具
sudo apt install htop tmux tree jq

# tmux配置
# ~/.tmux.conf
set -g mouse on
set -g base-index 1
set -g pane-base-index 1
```

**Docker开发环境**

```yaml
# docker-compose.yml

services:
  web:
    build: .
    ports:
      - "3000:3000"

      - .:/app
```

**环境变量管理**

```bash

load_dotenv()

# Node.js
require('dotenv').config()
```

## 环境搭建最佳实践

开发环境搭建的一些最佳实践。

**标准化环境**

1. 使用容器化环境
2. 编写环境搭建脚本
3. 使用配置管理工具
4. 文档化环境要求

**依赖管理**

1. 锁定依赖版本
2. 定期更新依赖
3. 使用私有仓库
4. 安全扫描依赖

**版本控制**

1. .gitignore中排除敏感文件
2. 不要提交IDE配置
3. 使用editorconfig统一编辑器设置
4. 提供环境搭建文档

**自动化**

```bash
# setup.sh
#!/bin/bash
set -e

# 安装依赖
sudo apt update
sudo apt install -y python3-dev build-essential

# 创建虚拟环境
python3 -m venv .venv
source .venv/bin/activate

# 安装Python依赖
pip install -r requirements.txt

# 安装前端依赖
npm install

echo "环境搭建完成！"
```

**持续集成**

在CI环境中也使用相同的环境配置，确保一致性。

开发环境的搭建要规范化，最好写成文档或脚本，方便团队成员快速搭建。
