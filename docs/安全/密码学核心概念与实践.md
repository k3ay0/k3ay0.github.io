# 密码学核心概念与实践：从对称加密到数字签名

密码学是信息安全的基石。无论是日常使用的HTTPS，还是区块链中的数字签名，背后都是密码学技术在支撑。

本文将系统梳理密码学的核心概念，包括对称加密、非对称加密、哈希函数和数字签名，并通过代码示例帮助理解这些抽象的概念。

## 对称加密：高效但需要安全的密钥分发

对称加密的特点是加密和解密使用相同的密钥。速度很快，适合处理大量数据。

**AES算法**

AES是目前最流行的对称加密算法，密钥长度可以是128、192或256位。

```python
from Crypto.Cipher import AES
from Crypto.Random import get_random_bytes
from Crypto.Util.Padding import pad, unpad

# 加密
key = get_random_bytes(32)  # 256位密钥
cipher = AES.new(key, AES.MODE_CBC)
plaintext = b'Hello, World!'
ciphertext = cipher.encrypt(pad(plaintext, AES.block_size))

# 解密
cipher = AES.new(key, AES.MODE_CBC, iv=cipher.iv)
plaintext = unpad(cipher.decrypt(ciphertext), AES.block_size)
```

**工作模式**

AES支持多种工作模式：
- ECB：简单但不安全，相同明文产生相同密文
- CBC：需要初始化向量，安全性更好
- CTR：支持并行处理，效率高
- GCM：提供认证加密，同时保证机密性和完整性

**DES算法**

DES是早期的标准，密钥只有56位，已经不安全了。了解它主要是为了理解密码学的发展历程。

3DES是DES的改进版本，使用三次加密来增强安全性，但效率较低，已逐渐被AES取代。

## 非对称加密：解决密钥分发问题

非对称加密使用一对密钥：公钥加密，私钥解密。解决了密钥分发的安全问题。

**RSA算法**

RSA是最著名的非对称加密算法，安全性基于大数分解的困难性。

密钥生成步骤：
1. 选择两个大素数p和q
2. 计算n = p * q
3. 计算欧拉函数φ(n) = (p-1)(q-1)
4. 选择e，使得gcd(e, φ(n)) = 1
5. 计算d，使得e*d ≡ 1 (mod φ(n))

公钥是(e, n)，私钥是(d, n)。

```python
from Crypto.PublicKey import RSA
from Crypto.Cipher import PKCS1_OAEP

# 生成密钥对
key = RSA.generate(2048)
public_key = key.publickey()

# 加密
cipher = PKCS1_OAEP.new(public_key)
ciphertext = cipher.encrypt(b'Hello, World!')

# 解密
cipher = PKCS1_OAEP.new(key)
plaintext = cipher.decrypt(ciphertext)
```

**椭圆曲线密码学**

ECC相比RSA，相同安全强度下密钥更短，计算效率更高。

```
RSA 2048位 ≈ ECC 256位
```

在移动设备和物联网场景中，ECC的优势明显。比特币和以太坊都使用ECDSA（椭圆曲线数字签名算法）。

## 哈希函数：数据完整性的守护者

哈希函数将任意长度的输入映射为固定长度的输出，具有单向性和抗碰撞性。

**SHA系列**

```python
import hashlib

# SHA-256
data = b'Hello, World!'
sha256_hash = hashlib.sha256(data).hexdigest()
print(f"SHA-256: {sha256_hash}")

# SHA-512
sha512_hash = hashlib.sha512(data).hexdigest()
print(f"SHA-512: {sha512_hash}")
```

**密码学哈希的特性**

- 单向性：无法从哈希值反推原数据
- 抗碰击：很难找到两个不同的输入产生相同的哈希值
- 雪崩效应：输入的微小变化导致输出的巨大变化

**应用**

1. 密码存储：不存储明文密码，只存储哈希值
2. 数据完整性：验证文件是否被篡改
3. 数字签名：先哈希再签名，提高效率
4. 区块链：工作量证明的核心

**SHA-1的衰落**

SHA-1曾是广泛使用的哈希算法，但已被证明存在碰撞攻击。Google在2017年展示了第一个SHA-1碰撞实例。

现在应该使用SHA-256或更高版本。

## 数字签名：认证与不可否认性

数字签名提供了认证、完整性和不可否认性。

**工作原理**

1. 发送方用哈希函数计算消息的摘要
2. 用自己的私钥加密摘要，形成签名
3. 接收方用发送方的公钥解密签名，得到摘要
4. 接收方自己计算消息的摘要，与解密得到的比较

```python
from Crypto.PublicKey import RSA
from Crypto.Signature import pkcs1_15
from Crypto.Hash import SHA256

# 生成密钥对
key = RSA.generate(2048)

# 签名
message = b'Hello, World!'
h = SHA256.new(message)
signature = pkcs1_15.new(key).sign(h)

# 验证
try:
    pkcs1_15.new(key.publickey()).verify(h, signature)
    print("签名验证成功")
except (ValueError, TypeError):
    print("签名验证失败")
```

**数字证书**

数字证书是数字签名的一种应用，由CA（证书颁发机构）签发，用于证明公钥的所有者。

HTTPS的工作流程：
1. 服务器发送数字证书
2. 浏览器验证证书的有效性
3. 浏览器用证书中的公钥加密会话密钥
4. 服务器用私钥解密，获得会话密钥
5. 双方用会话密钥进行加密通信

## 实践中的常见问题

密码学理论不难，但实践中有很多细节要注意。

**密钥管理**

密码学的安全性很大程度上取决于密钥管理。常见的问题：
- 密钥硬编码在代码中
- 密钥存储不安全
- 密钥轮换机制缺失

建议使用专门的密钥管理系统（如AWS KMS、HashiCorp Vault）。

**随机数生成**

密码学对随机数的质量要求很高。伪随机数生成器如果可预测，整个系统就不安全。

```python
import secrets  # 密码学安全的随机数

# 生成随机密钥
key = secrets.token_bytes(32)

# 生成随机数
random_number = secrets.randbelow(1000)
```

**填充攻击**

RSA的PKCS#1填充如果实现不当，可能遭受选择密文攻击。使用OAEP填充模式更安全。

**侧信道攻击**

即使算法本身安全，实现方式可能泄露信息。比如通过执行时间推测密钥。使用常数时间的实现可以缓解这类攻击。

**总结**

密码学是一门理论和实践并重的学科。选择合适的算法、正确实现、妥善管理密钥，三者缺一不可。
